Milestone 2 changes and run record
Date: 2026-03-29

Implementation changes
- Added a new hybrid benchmark adapter in benchmarks/benchmark_hybrid_pgm_lipp.h and benchmarks/benchmark_hybrid_pgm_lipp.cc.
- Added competitors/hybrid_pgm_lipp.h implementing a naive hybrid DPGM + LIPP index:
  - Bulk-load initial keys into LIPP.
  - Insert all new keys into DynamicPGM.
  - On lookup, check DynamicPGM first, then LIPP.
  - When the staged DynamicPGM size reaches a threshold, flush every staged key into LIPP with individual LIPP inserts, then reset the DynamicPGM buffer.
- Registered HybridPGMLIPP in benchmark.cc and CMakeLists.txt.
- Updated benchmark.h so CSV output can be redirected with TLI_RESULTS_DIR.
- Patched competitors/PGM-index/include/pgm_index_dynamic.hpp:
  - Fixed begin() iterator construction.
  - Added for_each(...) so the hybrid can flush staged keys without auxiliary data structures.
